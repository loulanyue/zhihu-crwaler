# 安全政策（Security Policy）

感谢您帮助维护 zhihu-crwaler 的安全。

---

## 支持的版本

我们仅对 `master` 分支最新版本提供安全修复支持。

| 版本 | 支持状态 |
|---|---|
| 最新 `master` 分支 | ✅ 持续支持 |
| 旧版本 | ❌ 不再支持 |

---

## 报告安全漏洞

**请勿通过公开的 GitHub Issues 报告安全漏洞。**

如果您发现任何安全漏洞，请通过以下方式进行负责任披露：

- 通过 [GitHub Security Advisories](https://github.com/loulanyue/zhihu-crwaler/security/advisories/new) 私密报告
- 或发送邮件至项目维护者

### 报告时请包含以下信息

- 漏洞类型（如：依赖注入、路径遍历、不安全的 HTTP 请求等）
- 漏洞所在的源文件路径（含行号或方法名）
- 受影响的版本（commit SHA 或 tag）
- 最小化复现步骤
- 漏洞影响评估

---

## 处理流程与时效

1. **确认收到**：在收到报告后 **3 个工作日**内发送确认回复
2. **初步评估**：在 **7 个工作日**内完成影响评估并告知结论
3. **修复与发布**：根据漏洞严重程度，修复时间从数小时到数周不等
4. **公开披露**：修复版本发布后，在 `CHANGELOG.md` 中注明安全修复条目
